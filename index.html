Comandos Básicos

Estos son los comandos fundamentales que todo desarrollador debe conocer.

git init

Inicializa un nuevo repositorio de Git en el directorio actual.

git init

git config

Configura tu identidad en Git (nombre y email).

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

git status

Muestra el estado actual de los archivos en el área de staging.

git status

git add

Agrega archivos al área de staging para el próximo commit.

git add archivo.txt
git add . # Agrega todos los archivos

git commit

Guarda los cambios del área de staging en el historial del repositorio.

git commit -m "Descripción del commit"
git commit -am "Commit directo" # Solo archivos modificados

git log

Muestra el historial de commits del repositorio.

git log
git log --oneline # Formato compacto
git log --graph --all --decorate # Vista gráfica

Comandos Modernos (Git 2.23+)

Estos comandos modernos reemplazan algunos usos de git checkout para mayor claridad.

git switch

Cambia entre ramas de forma más intuitiva que git checkout.

git switch main # Cambiar a rama main
git switch -c nueva-rama # Crear y cambiar
git switch - # Volver a la rama anterior

git restore

Restaura archivos del working directory o staging area.

git restore archivo.txt # Descartar cambios
git restore --staged archivo.txt # Quitar del staging
git restore --source=HEAD~1 archivo.txt # Desde commit específico

⚠️ Comandos Importantes (Usar con Precaución)

Estos comandos pueden modificar el historial o eliminar cambios permanentemente.

git reset

Mueve el HEAD y puede modificar el staging area y working directory.

git reset --soft HEAD~1 # Mantiene cambios en staging
git reset --mixed HEAD~1 # Mantiene cambios en working dir
git reset --hard HEAD~1 # ⚠️ ELIMINA CAMBIOS
⚠️ Advertencia: git reset --hard elimina cambios permanentemente. Úsalo solo como último recurso.

git reflog

Muestra el historial completo de referencias, incluso commits "eliminados".

git reflog
git reflog --all # Todas las referencias

Trabajo con Ramas

Las ramas permiten desarrollar funcionalidades de forma aislada.

git branch

Gestiona las ramas del repositorio.

git branch # Listar ramas locales
git branch -a # Todas las ramas
git branch nueva-rama # Crear rama
git branch -d rama # Eliminar rama

git merge

Fusiona cambios de una rama a otra.

git merge feature-branch
git merge --no-ff feature-branch # Preserva historial
git merge --squash feature-branch # Combina commits

Resolución de Conflictos

Cuando Git no puede fusionar automáticamente los cambios.

git status # Ver archivos en conflicto
git add archivo-resuelto.txt
git commit # Finalizar merge
💡 Tip: Los editores modernos como VS Code muestran herramientas visuales para resolver conflictos.

Repositorios Remotos

Comandos para trabajar con repositorios en GitHub, GitLab, etc.

git remote

Gestiona conexiones a repositorios remotos.

git remote -v # Ver remotos configurados
git remote add origin https://github.com/usuario/repo.git
git remote set-url origin nueva-url

git push

Envía commits locales al repositorio remoto.

git push origin main
git push -u origin feature-branch # Primera vez
git push --tags # Enviar tags

git pull

Descarga y fusiona cambios del repositorio remoto.

git pull origin main
git pull --rebase # Evita merge commits
git pull --allow-unrelated-histories # Historias diferentes

git fetch

Descarga cambios sin fusionar automáticamente.

git fetch origin
git fetch --all # Todos los remotos
git fetch --prune # Limpia referencias obsoletas

Configuración SSH

Configura autenticación segura con llaves SSH.

1. Generar llaves SSH

ssh-keygen -t ed25519 -C "tu@email.com"
# Para sistemas antiguos: ssh-keygen -t rsa -b 4096 -C "tu@email.com"

2. Iniciar el agente SSH

eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

3. Copiar llave pública

cat ~/.ssh/id_ed25519.pub
# Copia el contenido y agrégalo en GitHub → Settings → SSH Keys

4. Probar conexión

ssh -T git@github.com

🔐 Firmas GPG y Verificación de Commits

Firma tus commits con GPG para verificar tu identidad y aumentar la seguridad.

1. Generar llave GPG

gpg --full-generate-key
# Selecciona: RSA (4096 bits), sin expiración
# Usa el mismo email que en Git
💡 Configuración recomendada:
• Tipo: RSA and RSA (default)
• Tamaño: 4096 bits
• Expiración: 0 (no expira) o 2 años
• Email: El mismo configurado en Git

2. Listar y obtener ID de la llave

gpg --list-secret-keys --keyid-format=long
# Copia el ID después de "sec rsa4096/"
gpg --armor --export [KEY_ID] # Exportar llave pública

3. Configurar Git para usar GPG

git config --global user.signingkey [KEY_ID]
git config --global commit.gpgsign true # Firmar automáticamente
git config --global gpg.program gpg # Especificar programa GPG

4. Agregar llave GPG a GitHub

gpg --armor --export [KEY_ID] | pbcopy # macOS
gpg --armor --export [KEY_ID] | xclip -sel clip # Linux
# Ve a GitHub → Settings → SSH and GPG keys → New GPG key

Comandos de Firma

Firmar commits y tags manualmente.

git commit -S -m "Commit firmado" # Commit firmado
git tag -s v1.0.0 -m "Versión firmada" # Tag firmado
git log --show-signature # Ver firmas en el log

Verificación de Firmas

Verificar la autenticidad de commits y tags.

git verify-commit HEAD # Verificar último commit
git verify-tag v1.0.0 # Verificar tag
git log --pretty="format:%h %G? %aN %s" # Estado de firmas
🔍 Estados de firma:
• G: Firma válida
• B: Firma inválida
• U: Firma válida, confianza desconocida
• X: Firma expirada
• N: Sin firma

Gestión de Llaves GPG

Administrar y mantener tus llaves GPG.

gpg --list-keys # Listar llaves públicas
gpg --delete-key [KEY_ID] # Eliminar llave pública
gpg --delete-secret-key [KEY_ID] # Eliminar llave privada
gpg --export-secret-keys > backup.gpg # Backup de llaves
⚠️ Importante: Haz backup de tus llaves privadas y guárdalas en un lugar seguro. Si las pierdes, no podrás firmar commits con esa identidad.

Configuración Avanzada

Opciones adicionales para mejorar la experiencia con GPG.

echo 'export GPG_TTY=$(tty)' >> ~/.bashrc # Bash
echo 'export GPG_TTY=$(tty)' >> ~/.zshrc # Zsh
git config --global tag.gpgsign true # Firmar tags automáticamente

🏆 Mejores Prácticas con GPG

🔐 Seguridad

  • Usa una passphrase fuerte para tu llave privada
  • Haz backup de tus llaves en un lugar seguro
  • Considera usar una fecha de expiración (1-2 años)
  • Revoca llaves comprometidas inmediatamente

⚙️ Configuración

  • Usa el mismo email en Git y GPG
  • Configura GPG_TTY para evitar errores
  • Habilita firma automática para commits y tags
  • Verifica que GitHub reconoce tu llave

🔄 Mantenimiento

  • Renueva llaves antes de que expiren
  • Actualiza la llave en GitHub si cambias
  • Mantén actualizado el software GPG
  • Documenta el proceso para tu equipo

👥 Colaboración

  • Establece políticas de firma en el equipo
  • Usa branch protection rules en GitHub
  • Educa al equipo sobre verificación
  • Considera usar llaves de organización

Tags y Versiones

Marca versiones importantes de tu proyecto.

git tag

Crea y gestiona etiquetas de versión.

git tag v1.0.0 # Tag ligero
git tag -a v1.0.0 -m "Versión 1.0.0" # Tag anotado
git tag -l "v1.*" # Listar tags

Versionado Semántico

Sigue el estándar MAJOR.MINOR.PATCH (ej: 2.1.3).

git tag -a v2.1.3 -m "Fix: Corrección de bug crítico"
git push origin v2.1.3 # Publicar tag
git push origin --tags # Todos los tags
📋 Versionado Semántico:
• MAJOR: Cambios incompatibles
• MINOR: Nueva funcionalidad compatible
• PATCH: Correcciones de bugs

Comandos Avanzados

Herramientas poderosas para flujos de trabajo complejos.

git stash

Guarda temporalmente cambios sin hacer commit.

git stash # Guardar cambios
git stash pop # Aplicar y eliminar
git stash list # Ver stashes
git stash apply stash@{0} # Aplicar específico

git rebase

Reescribe el historial para un flujo más limpio.

git rebase main # Rebase sobre main
git rebase -i HEAD~3 # Rebase interactivo
git rebase --continue # Continuar después de resolver conflictos
⚠️ Advertencia: No hagas rebase de commits que ya están en repositorios públicos.

git bisect

Encuentra el commit que introdujo un bug usando búsqueda binaria.

git bisect start
git bisect bad # Commit actual tiene el bug
git bisect good v1.0.0 # Versión sin bug
git bisect reset # Terminar bisect

git worktree

Trabaja con múltiples ramas simultáneamente.

git worktree add ../feature-branch feature
git worktree list # Ver worktrees
git worktree remove ../feature-branch

git blame

Muestra quién modificó cada línea de un archivo.

git blame archivo.js
git blame -L 10,20 archivo.js # Líneas específicas
git blame -C archivo.js # Detecta copias

git cherry-pick

Aplica commits específicos a la rama actual.

git cherry-pick abc123
git cherry-pick abc123..def456 # Rango de commits
git cherry-pick --no-commit abc123 # Sin commit automático

Git Hooks

Automatiza tareas con scripts que se ejecutan en eventos específicos.

Hooks Comunes

Scripts que se ejecutan automáticamente en ciertos eventos.

ls .git/hooks/ # Ver hooks disponibles
chmod +x .git/hooks/pre-commit # Hacer ejecutable
🔧 Hooks útiles:
• pre-commit: Ejecutar tests antes del commit
• pre-push: Validaciones antes del push
• post-merge: Instalar dependencias después del merge

Ejemplo: Pre-commit Hook

Script que ejecuta linting antes de cada commit.

#!/bin/sh
# .git/hooks/pre-commit
npm run lint
if [ $? -ne 0 ]; then
echo "Linting failed. Commit aborted."
exit 1
fi

GitHub Actions Básico

Automatización de CI/CD directamente en GitHub.

Estructura Básica

Archivo de workflow en .github/workflows/

# .github/workflows/ci.yml
name: CI
on: [push, pull_request]
jobs:
test:
runs-on: ubuntu-latest
steps:
- uses: actions/checkout@v3
- run: npm test

Comandos Útiles

Gestión de workflows desde la línea de comandos.

gh workflow list # Listar workflows
gh workflow run ci.yml # Ejecutar workflow
gh run list # Ver ejecuciones
💡 Tip: Instala GitHub CLI para gestionar GitHub desde la terminal.

🔧 Solución de Problemas Comunes

Soluciones a los problemas más frecuentes en Git.

❌ "Permission denied (publickey)"

Problema: Error de autenticación SSH

Solución:

ssh-add -l # Verificar llaves cargadas
ssh-add ~/.ssh/id_ed25519 # Agregar llave
ssh -T git@github.com # Probar conexión

❌ "Your branch is ahead of origin/main"

Problema: Commits locales no enviados al remoto

Solución:

git push origin main # Enviar commits
git pull --rebase # Si hay conflictos

❌ "Merge conflict"

Problema: Conflictos durante merge o pull

Solución:

git status # Ver archivos en conflicto
# Editar archivos y resolver conflictos
git add archivo-resuelto.txt
git commit # Finalizar merge

❌ "Detached HEAD state"

Problema: No estás en ninguna rama específica

Solución:

git branch temp-branch # Crear rama temporal
git switch main # Volver a main
git merge temp-branch # Si quieres conservar cambios

❌ "File too large" (GitHub)

Problema: Archivo mayor a 100MB

Solución:

git lfs track "*.zip" # Usar Git LFS
git add .gitattributes
git add archivo-grande.zip
git commit -m "Add large file with LFS"

❌ Commit accidental

Problema: Hiciste commit de algo que no querías

Solución:

git reset --soft HEAD~1 # Deshacer último commit
git restore --staged archivo.txt # Quitar del staging
git commit -m "Commit corregido"

❌ "gpg: signing failed: Inappropriate ioctl for device"

Problema: Error al firmar commits con GPG

Solución:

export GPG_TTY=$(tty) # Configurar terminal
echo 'export GPG_TTY=$(tty)' >> ~/.bashrc # Permanente
gpg-connect-agent reloadagent /bye # Reiniciar agente

❌ "No secret key" al firmar

Problema: Git no encuentra la llave GPG configurada

Solución:

gpg --list-secret-keys --keyid-format=long # Ver llaves
git config --global user.signingkey [KEY_ID] # Configurar llave
git config --global user.email # Verificar email

📋 Referencia Rápida

Los comandos más utilizados en el día a día.

Básicos

  • git status - Estado actual
  • git add . - Agregar todos los archivos
  • git commit -m "mensaje" - Hacer commit
  • git push - Enviar cambios
  • git pull - Recibir cambios

Ramas

  • git branch - Listar ramas
  • git switch rama - Cambiar rama
  • git switch -c nueva - Crear y cambiar
  • git merge rama - Fusionar rama
  • git branch -d rama - Eliminar rama

Historial

  • git log --oneline - Historial compacto
  • git log --graph - Vista gráfica
  • git show commit - Ver commit específico
  • git diff - Ver diferencias
  • git blame archivo - Quién cambió qué

Deshacer

  • git restore archivo - Descartar cambios
  • git restore --staged archivo - Quitar del staging
  • git reset --soft HEAD~1 - Deshacer commit
  • git revert commit - Revertir commit
  • git reflog - Historial completo

🏆 Mejores Prácticas

📝 Mensajes de Commit

  • Usa el presente imperativo: "Add feature" no "Added feature"
  • Primera línea máximo 50 caracteres
  • Usa prefijos: feat:, fix:, docs:, style:, refactor:
  • Explica el "qué" y "por qué", no el "cómo"

🌿 Estrategia de Ramas

  • main/master: código estable en producción
  • develop: integración de nuevas funcionalidades
  • feature/nombre: desarrollo de funcionalidades
  • hotfix/nombre: correcciones urgentes

🔒 Seguridad

  • Nunca hagas commit de credenciales o API keys
  • Usa .gitignore para archivos sensibles
  • Configura llaves SSH en lugar de HTTPS
  • Revisa los cambios antes de hacer push

⚡ Flujo de Trabajo

  • Haz commits pequeños y frecuentes
  • Siempre haz pull antes de push
  • Usa ramas para cada funcionalidad
  • Revisa el código antes de hacer merge
